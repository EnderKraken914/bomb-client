# Bomb Client 2.0 Core

Bomb Client 2.0 Core turns the launcher into a safer external client framework for Minecraft Bedrock: modules, server profiles, managed packs, bridge-ready overlay data, and a proper update hub.

## Highlights

- Internal module architecture for overlays, visuals, account, profiles, servers, packs, bridge, updates, and future features.
- Server profiles for public Bedrock servers using only safe external data: saved host/port, notes, favorites, categories, presets, and normal ping/status checks.
- Packs tab for Bomb Client-managed `.mcpack`, `.mcaddon`, and `.zip` imports without editing Minecraft's installed game files.
- Bomb Server Bridge foundation for cooperating servers, disabled by default and mockable for testing.
- Latest Updates tab connected to the update manifest metadata.
- Self-test mode for core systems.

## Safety Statement

Bomb Client 2.0 remains external only. It does not inject into Minecraft Bedrock, edit memory, hook game processes, modify packets, sniff traffic, bypass anti-cheat, reverse engineer Bedrock internals, or alter Microsoft Store Minecraft installation files.

## Known Limitations

Advanced live game data from public servers is unavailable unless the server owner intentionally supports a Bomb Client-compatible bridge. Public servers still work with external overlays, profiles, notes, local stats, safe ping checks, and user presets.
